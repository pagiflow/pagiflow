# Contributing to Pagiflow

First off, thank you for considering contributing to Pagiflow! It's people like you that make Pagiflow such a great tool for everyone.

## How Can I Contribute?

### Reporting Bugs
* Check the existing issues to see if the bug has already been reported.
* If not, open a new issue. Include a clear description, steps to reproduce, and any relevant code snippets.

### Suggesting Enhancements
* Open an issue describing the feature you'd like to see.
* Explain why it would be useful and how it should work.

### Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or bug fix.
3. Commit your changes with clear, descriptive messages.
4. Push your branch to your fork.
5. Submit a pull request.

## Development Setup
1. Clone your fork.
2. Open `index.html` in your browser to see the demo.
3. Make changes to the files in the `src/` directory.
4. If you add a new feature, please update the documentation or add a demo in `pagiflow-demos.html`.

## Code Style
* Use clear, descriptive variable and function names.
* Comment your code where necessary.
* Ensure your code is compatible with modern browsers.

Thank you for your contributions!
